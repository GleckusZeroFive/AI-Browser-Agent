# Changelog

Все значимые изменения в проекте будут документированы в этом файле.

## [Unreleased] - 2026-01-15

### Добавлено

#### Системная проверка при запуске
- Новый модуль `src/system_check.py` для проверки системных требований
- Проверка версии Python (требуется 3.10+)
- Проверка установки Playwright
- Проверка установки Firefox браузера для Playwright
- Проверка наличия и корректности файла .env
- Проверка установки всех зависимостей
- Автоматический запуск проверок при старте приложения

#### Вывод версий библиотек
- Новый модуль `src/version_info.py` для отображения версий
- Вывод версий Python, OpenAI SDK, Playwright, python-dotenv при запуске
- Функция получения версий для логирования
- Опциональный verbose режим с дополнительной информацией

#### Улучшенные сообщения об ошибках
- **Browser Tools** ([src/tools/browser_tools.py](src/tools/browser_tools.py)):
  - Специфичные сообщения для ошибок запуска браузера
  - Детальные инструкции при отсутствии Firefox
  - Улучшенные сообщения для ошибок навигации (таймаут, сетевые ошибки)
  - Информативные сообщения при ошибках взаимодействия с элементами

- **AI Agent** ([src/agent/ai_agent.py](src/agent/ai_agent.py)):
  - Детальные сообщения при ошибках API (rate limit, авторизация, сеть)
  - Информативные подсказки по устранению проблем
  - Автоматическое переключение между моделями с уведомлениями
  - Специфичные сообщения для разных типов ошибок API

- **Config** ([src/config.py](src/config.py)):
  - Подробные инструкции при отсутствии GROQ_API_KEY
  - Пошаговое руководство по получению и настройке API ключа

#### Инфраструктура для тестирования
- Файл `requirements-dev.txt` с зависимостями для разработки:
  - pytest и плагины (asyncio, cov, mock, timeout, xdist)
  - Инструменты качества кода (flake8, black, mypy, pylint, isort)
  - Инструменты для отладки (ipdb, ipython)

- Конфигурация pytest (`pytest.ini`):

  - Маркеры для категоризации тестов (unit, integration, slow, browser, api)

- Общие фикстуры (`tests/conftest.py`):
  - Моки для браузера, AI агента, конфигурации
  - Фикстуры для реального браузера (интеграционные тесты)
  - Фикстуры для временных файлов и окружения
  - Автоматическая настройка окружения для тестов

- Unit тесты:
  - `tests/unit/test_system_check.py` - тесты системной проверки
  - `tests/unit/test_version_info.py` - тесты модуля версий

- Документация по тестированию (`tests/README.md`):
  - Инструкции по установке зависимостей
  - Примеры запуска различных типов тестов
  - Руководство по написанию тестов
  - Лучшие практики тестирования

## Использование

### Запуск приложения с новыми проверками

```bash
python main.py
```

При запуске вы увидите:
1. Информацию о версиях установленных библиотек
2. Результаты проверки системных требований
3. Детальные инструкции если что-то не настроено

### Запуск тестов

```bash
# Установить dev зависимости
pip install -r requirements-dev.txt

# Запустить все тесты
pytest

# Запустить только unit тесты
pytest -m unit

# Запустить с покрытием
pytest --cov=src --cov-report=html
```

## Breaking Changes

Нет breaking changes в этом релизе.

## Migration Guide

Для обновления до новой версии:

1. Установите обновленные зависимости:
   ```bash
   pip install -r requirements.txt
   ```

2. (Опционально) Установите dev зависимости для тестирования:
   ```bash
   pip install -r requirements-dev.txt
   ```

3. Убедитесь что Firefox установлен:
   ```bash
   playwright install firefox
   ```

4. Запустите приложение - системная проверка запустится автоматически:
   ```bash
   python main.py
   ```
