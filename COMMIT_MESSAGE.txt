fix: Восстановить память агента после оптимизации контекста

Проблема:
После оптимизации токенов агент терял контекст задачи, не помнил
показанные варианты и переспрашивал информацию.

Решение:

1. Добавлена краткосрочная память (working_memory) в Knowledge Base
   - current_task: текущая задача агента
   - current_page: где находится агент
   - shown_options: варианты которые показал пользователю
   - last_action: последнее действие

2. Исправлено пересоздание агента при смене задачи
   - Теперь обновляется только системный промпт
   - История диалога полностью сохраняется

3. Улучшен Context Compressor
   - Добавлен бонус за недавность (+0.75 для последних 5 сообщений)
   - Недавние сообщения гарантированно не удаляются

4. Автоматическое обновление working_memory
   - Извлекается из ответа агента через regex
   - Обновляется после каждого ответа

Результат:
Агент теперь помнит контекст и правильно реагирует на запросы типа
"Прочитай состав" после того как показал варианты.

Файлы:
- src/agent/knowledge_base.py: добавлена working_memory структура
- src/agent/context_compressor.py: добавлен recency bonus
- src/dialogue_manager.py: исправлено пересоздание агента
- src/agent/ai_agent.py: обновлены генераторы контекста
- FIXES_CONTEXT_MEMORY.md: полная документация изменений
