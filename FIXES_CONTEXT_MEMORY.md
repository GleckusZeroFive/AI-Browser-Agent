# Исправление проблем с памятью агента после оптимизации контекста

## 🔍 Проблема

После оптимизации контекста и токенов агент стал работать плохо:

1. **Терял контекст задачи** - не помнил что только что показывал варианты пиццы
2. **Переспрашивал информацию** - когда пользователь говорил "Прочитай состав", агент не понимал о чем речь
3. **Пересоздание агента** - при смене типа задачи агент терял всю историю диалога
4. **Слишком агрессивная компрессия** - удалялись недавние важные сообщения

### Пример из логов:

```
User: Хочу заказать пиццу Маргарита из Додопиццы
Agent: [Показывает 2 варианта Маргариты]

User: Прочитай состав
Agent: Чтобы прочитать состав, мне нужно знать о каком составе идет речь...  ❌

User: Состав блюд
Agent: Чтобы прочитать состав блюд, мне нужно перейти на страницу...  ❌
```

## ✅ Решение

### 1. Добавлена краткосрочная память (Working Memory) в Knowledge Base

**Файл:** `src/agent/knowledge_base.py`

Добавлен новый раздел `working_memory` в структуру базы знаний:

```python
"working_memory": {
    "current_task": None,       # "заказываю Маргариту из Додопиццы"
    "current_page": None,       # "на странице поиска Яндекс.Еды"
    "shown_options": [],        # ["Маргарита 490₽", "Маргарита с томатным соусом 520₽"]
    "last_action": None,        # "показал варианты пиццы"
    "context": {}               # произвольный контекст для текущей задачи
}
```

**Новые методы:**
- `set_working_context()` - обновить краткосрочную память
- `get_working_context()` - получить текущий контекст
- `clear_working_memory()` - очистить при завершении задачи

### 2. Обновлены генераторы контекста

Working memory теперь включается во **все** уровни контекста:

#### MINIMAL (~150 токенов вместо 100):
```
🎯 ТЕКУЩАЯ ЗАДАЧА: заказываю Маргариту из Додопиццы
📄 ГДЕ ТЫ: на странице поиска/результатов
📋 ПОКАЗАЛ ВАРИАНТЫ: Маргарита 490₽, Маргарита с томатным соусом 520₽
📍 Местоположение: Красноярск
⚠️  Аллергии: нет
```

#### COMPACT (~400 токенов вместо 300):
Включает working memory + базовую информацию + предпочтения

#### FULL (~1000+ токенов):
Включает working memory + полную базу знаний

### 3. Исправлено пересоздание агента

**Файл:** `src/dialogue_manager.py`

**БЫЛО:**
```python
if task_type != self.current_task_type:
    # Пересоздаём AI агента с новым промптом
    self.agent = AIAgent()  # ❌ Теряет всю историю!
```

**СТАЛО:**
```python
if task_type != self.current_task_type:
    # ИСПРАВЛЕНИЕ: НЕ пересоздаём агента! Только обновляем системный промпт
    # Это сохраняет всю историю разговора
    system_prompt = specialized_agent.get_system_prompt()
    self.agent._update_system_message(system_prompt)  # ✅ История сохраняется
```

### 4. Улучшен Context Compressor

**Файл:** `src/agent/context_compressor.py`

Добавлен **бонус за недавность** - последние 5 сообщений получают приоритет:

```python
def _calculate_importance(self, message, msg_type, position_from_end):
    # НОВОЕ: Недавность - очень важный фактор!
    if position_from_end < 5:
        recency_bonus = (5 - position_from_end) * 0.15  # 0.75, 0.60, 0.45, 0.30, 0.15
        score += recency_bonus
```

**Результат:**
- Последнее сообщение: +0.75 к важности
- 2-е с конца: +0.60
- 3-е с конца: +0.45
- и т.д.

Это гарантирует что недавние сообщения не будут удалены при компрессии.

### 5. Автоматическое обновление working_memory

**Файл:** `src/dialogue_manager.py`

Добавлен новый метод `_update_working_memory_from_response()`, который:

1. **Анализирует ответ агента** и автоматически извлекает:
   - Текущую задачу (из reasoning или action)
   - Текущую страницу (из navigate/search/click)
   - Показанные варианты (нумерованные списки)
   - Последнее действие

2. **Вызывается автоматически** после каждого ответа агента:
```python
action = self.agent.parse_action(response)
# НОВОЕ: Автоматически обновляем working_memory
await self._update_working_memory_from_response(user_input, response, action)
```


## 🎯 Ключевые улучшения

1. **Краткосрочная память** - агент помнит что только что делал
2. **Сохранение истории** - переключение агента не стирает контекст
3. **Приоритет недавним** - последние сообщения не удаляются
4. **Автоматическое обновление** - working_memory обновляется без промптов

## 🔧 Технические детали

### Размеры контекста (токены):

| Уровень | Было | Стало | Причина |
|---------|------|-------|---------|
| MINIMAL | ~100 | ~150  | + working memory (3 строки) |
| COMPACT | ~300 | ~400  | + working memory (5-10 строк) |
| FULL    | ~800 | ~1000 | + working memory (полная) |

### Приоритет сообщений в Compressor:

```
Позиция с конца | Бонус | Итого (с базовой оценкой 0.3)
----------------|-------|--------------------------------
Последнее (#1)  | +0.75 | 1.05 → 1.0 (max)
2-е (#2)        | +0.60 | 0.90
3-е (#3)        | +0.45 | 0.75
4-е (#4)        | +0.30 | 0.60
5-е (#5)        | +0.15 | 0.45
6+ (#6+)        | +0.00 | 0.30 (базовая)
```

## 📝 Миграция данных

При первом запуске после обновления:
- Существующие knowledge_base.json автоматически получат новую структуру
- `working_memory` будет добавлена с пустыми значениями
- Версия обновится с `1.0` → `1.1`
