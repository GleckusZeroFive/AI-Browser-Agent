## С чего начать

### 1. Настройка окружения

Склонируйте репозиторий и настройте окружение разработки:

```bash
git clone <repository-url>
cd ai-browser-agent
python -m venv venv
source venv/bin/activate  # Linux/macOS
# или
venv\Scripts\activate     # Windows

pip install -r requirements.txt
pip install -r requirements-dev.txt
playwright install firefox
```

### 2. Настройка переменных окружения

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Добавьте ваш API ключ Groq в файл `.env`:

```env
GROQ_API_KEY=your_groq_api_key_here
```

### 3. Проверка установки

Запустите тесты, чтобы убедиться, что все работает:

```bash
pytest tests/
```

## Процесс разработки

### Выбор задачи

1. Посмотрите открытые [Issues](../../issues)
2. Issues с метками `good first issue` или `help wanted` отлично подходят для новичков
3. Если хотите работать над задачей, оставьте комментарий в Issue

### Создание ветки

Используйте описательные имена веток:

```bash
git checkout -b feature/add-new-browser-tool
git checkout -b fix/browser-crash-on-modal
git checkout -b docs/update-installation-guide
```

Формат: `type/short-description`

Типы:
- `feature/` - новая функциональность
- `fix/` - исправление бага
- `docs/` - изменения в документации
- `refactor/` - рефакторинг кода
- `test/` - добавление или изменение тестов

### Разработка

1. Пишите чистый, читаемый код
2. Следуйте стандартам проекта (см. ниже)
3. Добавляйте тесты для новой функциональности
4. Обновляйте документацию при необходимости

## Стандарты кода

### Форматирование Python

Проект следует [PEP 8](https://pep8.org/) с небольшими модификациями:

- Максимальная длина строки: 100 символов
- Используйте 4 пробела для отступов
- Используйте snake_case для функций и переменных
- Используйте PascalCase для классов

### Линтинг и форматирование

Перед коммитом запустите:

```bash
# Автоформатирование
black src/ tests/

# Проверка линтером
flake8 src/ tests/

# Проверка типов
mypy src/
```

## Тестирование

### Запуск тестов

```bash
# Все тесты
pytest

# Конкретный модуль
pytest tests/test_browser_tools.py

# С покрытием кода
pytest --cov=src tests/
```

### Написание тестов

- Добавляйте unit тесты для новых функций
- Интеграционные тесты для комплексных сценариев
- Используйте моки для внешних зависимостей (API, браузер)

Пример теста:

```python
def test_browser_navigation():
    """Тест навигации браузера по URL"""
    browser_tool = BrowserTool()
    result = browser_tool.navigate("https://example.com")
    assert result.success is True
    assert "example.com" in result.url
```

### Структура тестов

```
tests/
├── unit/              # Unit тесты
│   ├── test_agent.py
│   └── test_tools.py
├── integration/       # Интеграционные тесты
│   └── test_full_flow.py
└── fixtures/          # Тестовые данные
    └── mock_data.py
```

## Коммиты и Pull Request

### Сообщения коммитов

Используйте понятные сообщения коммитов на русском или английском:

```bash
git commit -m "feat: добавлена поддержка Chrome браузера"
git commit -m "fix: исправлен краш при работе с модальными окнами"
git commit -m "docs: обновлены инструкции по установке"
```

Формат: `type: краткое описание`

Типы:
- `feat` - новая функциональность
- `fix` - исправление бага
- `docs` - изменения в документации
- `refactor` - рефакторинг кода
- `test` - добавление/изменение тестов
- `chore` - рутинные задачи (обновление зависимостей и т.д.)

### Создание Pull Request

1. Убедитесь, что все тесты проходят
2. Обновите [CHANGELOG.md](CHANGELOG.md) если нужно
3. Запушьте ветку в ваш fork
4. Создайте Pull Request в основной репозиторий

**Описание PR должно включать:**

- Краткое описание изменений
- Ссылка на связанный Issue (если есть)
- Скриншоты (для UI изменений)
- Чеклист проверки

Пример шаблона PR:

```markdown
## Описание
Добавлена поддержка работы с Chrome браузером.

## Связанные Issues
Fixes #123

## Изменения
- Добавлен ChromeBrowserTool
- Обновлена документация
- Добавлены тесты

## Чеклист
- [x] Тесты проходят
- [x] Код отформатирован
- [x] Документация обновлена
- [x] CHANGELOG обновлен
```
