# Исправления: Exit команды и действие "dialog"

**Дата:** 2026-01-14
**Проблемы:** Агент не распознавал прощание в составе фраз, генерировал несуществующее действие "dialog"
**Статус:** ✅ Исправлено

## 🐛 Проблемы

### Проблема 1: "Спасибо. Пока" не распознаётся как exit

**Описание:**
Когда пользователь писал благодарность вместе с прощанием (например, "Спасибо. Пока"), агент НЕ завершал сессию, а тратил 40+ секунд на обработку через API.

**Пример из реальной сессии:**
```
👤 Пользователь: Спасибо. Пока
⏱️  17:51:06 - Сообщение отправлено
⏱️  17:51:50 - HTTP Request (44 секунды обработки!)
🤖 Агент ответил: ...
⏱️  17:51:53 - Завершение по команде пользователя
```

**Техническая причина:**
Код в [src/dialogue_manager.py:66](src/dialogue_manager.py#L66):
```python
# СТАРЫЙ КОД:
if user_input.lower() in ['exit', 'quit', 'выход', 'пока']:
```

Это проверяло **точное совпадение** строки, поэтому:
- ✅ `"пока"` → EXIT
- ❌ `"Спасибо. Пока"` → CONTINUE (не совпадает!)
- ❌ `"До свидания"` → CONTINUE (не в списке)

### Проблема 2: Действие "dialog" генерируется агентом

**Описание:**
Агент иногда генерировал `{"action": "dialog", "params": {"message": "..."}}` несмотря на запрет в промпте. ActionExecutor не поддерживал это действие → ошибка.

**Пример из логов:**
```
2026-01-14 17:49:07,338 [INFO] Выполняю действие: dialog
2026-01-14 17:49:07,338 [ERROR] Ошибка при выполнении действия: Неизвестное действие: dialog
```

**Реальный случай из agent_responses.log:**
```json
{
  "action": "dialog",
  "params": {
    "message": "Привет! Я помогу тебе заказать еду в Додо Пицце. Сколько человек будет?",
    "reasoning": "Начинаю диалог..."
  }
}
```

**Техническая причина:**
1. Промпт содержал **негативную инструкцию**: `НЕ используй {"action": "dialog"}`
2. LLM игнорируют негативные инструкции ("не делай X")
3. ActionExecutor не имел обработчика для `"dialog"` в `action_map`
4. Результат: агент ошибался, система падала

```python
def _is_exit_command(self, user_input: str) -> bool:
    """
    Проверить, является ли ввод пользователя командой выхода

    Использует word boundaries для корректного распознавания прощаний
    в составе фраз типа "Спасибо. Пока" и избегания ложных срабатываний
    на слова типа "Покажи" (содержит "пока")
    """
    text = user_input.lower().strip()

    # Однозначные команды выхода (английские и альтернативные русские)
    simple_exits = r'\b(exit|quit|выход|прощай|досвидания|довстречи|до\s*свидания|до\s*встречи)\b'
    if re.search(simple_exits, text, re.IGNORECASE):
        return True

    # "Пока" - только если НЕ в начале предложения с последующим "не"
    # Это отфильтрует "Пока не голоден", но оставит "Спасибо. Пока"
    if re.search(r'\bпока\b', text):
        # Проверяем: если "пока" в начале и за ним идёт "не" - это союз, не прощание
        if not re.match(r'^пока\s+не\b', text):
            return True

    return False
```

**Ключевые особенности:**
- ✅ Использует **word boundaries** (`\b`) для точного поиска слов
- ✅ Распознаёт фразы: "Спасибо. Пока", "Всё, пока", "До свидания"
- ✅ Игнорирует ложные срабатывания: "Покажи" (содержит "пока")
- ✅ Специальная логика для "пока не ..." (союз, не прощание)

### Фикс 2: Graceful fallback для "dialog"

#### Изменения в [src/agent/ai_agent.py](src/agent/ai_agent.py)

**Строки 96-109:** Улучшен промпт с позитивными инструкциями
```python
# БЫЛО (негативная инструкция):
НЕ используй {"action": "dialog"} - такого действия НЕТ!

# СТАЛО (позитивные + негативные):
ФОРМАТЫ ОТВЕТА:

ДИАЛОГ (нужна инфа) = ПРОСТО ТЕКСТ (БЕЗ JSON!)
"Привет" → "Привет! Что заказываем?"
"Пиццу" → "Сколько вас?"
"Что получилось?" → "Нашёл комбо '3 пиццы' за 929₽. Заказываем?"

ДЕЙСТВИЕ (выполняю работу) = ТОЛЬКО JSON
"Двое" → {"action": "navigate", "params": {"url": "..."}, "reasoning": "..."}

ВАЖНО:
✅ Для диалога → пиши ОБЫЧНЫЙ ТЕКСТ
❌ НЕ пиши {"action": "dialog"} - это действие НЕ СУЩЕСТВУЕТ!
✅ Если хочешь поговорить с пользователем → просто отвечай текстом БЕЗ JSON
```

**Почему это лучше:**
- Позитивные инструкции более понятны LLM ("делай X" вместо "не делай Y")
- Явные примеры правильного формата диалога
- Визуальные маркеры (✅/❌) усиливают важность

#### Изменения в [src/dialogue_manager.py](src/dialogue_manager.py)

**Строки 105-125:** Добавлена проверка и fallback для `"dialog"`
```python
if action:
    # Проверка на ошибочное использование "dialog"
    if action.get("action") == "dialog":
        # Graceful fallback: извлекаем текст и выводим как диалог
        self.logger.warning("Агент использовал action='dialog' (не существует). Извлекаю текст.")

        # Извлекаем текст из разных возможных полей
        dialog_text = (
            action.get("text") or
            action.get("params", {}).get("message") or
            action.get("params", {}).get("text") or
            "Извините, произошла внутренняя ошибка."
        )

        print(f"\n🤖 Агент: {dialog_text}\n")

    else:
        # Обычное действие - выполняем команду
        print(f"\n💭 Агент: {response.split('{')[0].strip()}")
        await self._execute_action_with_followup(action)
```

**Как это работает:**
1. Если агент всё же сгенерировал `{"action": "dialog", ...}`
2. Система НЕ бросает ошибку
3. Извлекается текст из `action.text` или `action.params.message`
4. Текст выводится пользователю как обычный диалог
5. Логируется warning для мониторинга
