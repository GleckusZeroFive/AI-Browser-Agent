# 📚 База знаний агента

## Описание

Агент теперь имеет **долговременную память** - базу знаний, которая сохраняется между сеансами работы. Это решает проблемы с повторяющимися вопросами и галлюцинациями.

## Что исправлено

### ❌ Было (старые проблемы):
1. **Агент переспрашивает одно и то же**
   - Пользователь: "Я в Красноярске"
   - Через 2 минуты агент: "В каком городе вы находитесь?"

2. **Агент галлюцинирует**
   - Агент: "Ресторан 'Итальянский Дворик' есть в Красноярске"
   - Реальность: такого ресторана нет

3. **Огромные логи**
   - Логи занимали сотни мегабайт
   - Нет автоматической очистки

### ✅ Стало (что улучшено):

1. **Долговременная память**
   ```
   USER: "Я в Красноярске"
   [База знаний: location = Красноярск]

   USER: "Хочу пиццу"
   AGENT: "Хорошо, поищу пиццу в Красноярске" ✅ НЕ переспрашивает!
   ```

2. **Проверка галлюцинаций**
   ```
   AGENT: "Ресторан 'Итальянский Дворик' есть в Красноярске"
   СИСТЕМА: ⚠️ ВНИМАНИЕ: Агент делает утверждение о существовании чего-либо.
            Попросите агента проверить эту информацию.
   ```

3. **Автоматическая ротация логов**
   - Максимум 5 MB на файл
   - Хранится только 3 последних файла
   - Старые логи (>7 дней) удаляются автоматически

## Структура базы знаний

База знаний хранится в `data/knowledge_base.json`:

```json
{
  "user_info": {
    "location": "Красноярск",
    "dietary_restrictions": ["аллергия на морепродукты"],
    "budget_preferences": {"food": 500},
    "preferences": {
      "food_types": ["итальянская кухня", "пицца"],
      "response_format": "краткий прогноз погоды"
    }
  },
  "verified_facts": {
    "restaurants": {
      "Красноярск": ["Додо Пицца", "Суши Wok"]
    },
    "services": {
      "доставка_еды": ["Яндекс.Еда", "Delivery Club"]
    }
  },
  "interaction_history": {
    "asked_questions": ["погода в Красноярске"],
    "confirmed_facts": ["любит итальянскую кухню"],
    "rejected_suggestions": ["японская кухня"]
  }
}
```

## Как это работает

### 1. Автоматическое извлечение информации

Агент **самостоятельно** извлекает и сохраняет информацию из диалога:

```python
USER: "Я в Красноярске, аллергия на морепродукты, бюджет 500 рублей"

# Система автоматически извлекает:
location = "Красноярск"
dietary_restrictions = ["аллергия на морепродукты"]
budget_preferences = {"food": 500}
```

### 2. Использование базы знаний в запросах

Каждый запрос к агенту автоматически дополняется информацией из базы:

```
📚 БАЗА ЗНАНИЙ О ПОЛЬЗОВАТЕЛЕ:
📍 Местоположение: Красноярск
🚨 Ограничения питания: аллергия на морепродукты
💰 Бюджет: food: 500₽

==================================================

USER: Хочу пиццу
```

### 3. Проверка галлюцинаций

Система анализирует ответы агента и предупреждает о потенциальных галлюцинациях:

```python
AGENT: "Ресторан 'Вкусняшка' есть в Красноярске"

# Система проверяет:
- Есть ли этот ресторан в verified_facts?
- Если НЕТ → предупреждение пользователю
```

## Настройки логирования

### Параметры ротации (в `src/utils/log_setup.py`):

```python
MAX_BYTES = 5 * 1024 * 1024  # 5 MB на файл
BACKUP_COUNT = 3              # Хранить 3 последних файла
LOG_RETENTION_DAYS = 7        # Удалять логи старше 7 дней
```

### Типы логов:

1. **dialogue_manager.log** - основные события диалога (INFO)
2. **errors.log** - только ошибки (ERROR)
3. **execution_YYYYMMDD.log** - детальное выполнение (DEBUG)
4. **agent_responses.log** - полные ответы агента

## API базы знаний

### Основные методы:

```python
# Инициализация
kb = KnowledgeBase(llm_client, storage_path="data/knowledge_base.json")

# Извлечение информации из диалога
await kb.extract_and_update(user_msg, agent_response)

# Получение резюме для передачи агенту
summary = kb.get_context_summary()

# Проверка существования факта
fact = kb.check_fact_exists("restaurant", "Итальянский Дворик")

# Определение нужна ли проверка утверждения
needs_check = kb.should_verify_before_claiming(agent_response)

# Добавление проверенного факта
kb.add_verified_fact("restaurants", "Красноярск", "Додо Пицца")

# Очистка базы знаний
kb.clear()
```

## Технические детали

### Архитектура

```
DialogueManager
    ↓
    ├── AIAgent (основной агент)
    ├── KnowledgeBase (долговременная память)
    │   ├── extract_and_update() - извлечение из диалога
    │   ├── get_context_summary() - резюме для агента
    │   └── should_verify_before_claiming() - детектор галлюцинаций
    └── ContextExtractor (краткосрочный контекст)
```

### Обработка запроса:

1. Пользователь вводит запрос
2. Система загружает резюме базы знаний
3. Формируется полный запрос: `[База знаний] + [Запрос пользователя]`
4. Агент отвечает с учётом базы знаний
5. Система извлекает новую информацию и обновляет базу
6. Проверяется наличие галлюцинаций
7. Ответ выводится пользователю (с предупреждением если нужно)

## Производительность

- **Размер базы знаний**: ~1-5 KB в JSON
- **Overhead на запрос**: +200-500 токенов (для передачи базы знаний)
- **Экономия**: Меньше переспросов = меньше запросов к API
- **Точность**: Резко снижает количество галлюцинаций

## Расширение

### Добавление новых типов информации:

1. Обновите структуру в `KnowledgeBase._load_or_create()`
2. Добавьте логику извлечения в промпт `extract_and_update()`
3. Добавьте форматирование в `get_context_summary()`

### Пример - добавление любимых блюд:

```python
# В _load_or_create()
"user_info": {
    ...
    "favorite_dishes": []
}

# В extraction_prompt
"favorite_dishes: блюда которые пользователь одобрил/понравились"

# В get_context_summary()
if prefs.get("favorite_dishes"):
    dishes = ", ".join(prefs["favorite_dishes"])
    lines.append(f"❤️  Любимые блюда: {dishes}")
```

## Решение проблем

### База знаний не сохраняется

Проверьте:
- Существует ли директория `data/`
- Есть ли права на запись
- Логи ошибок в `logs/errors.log`

### Агент игнорирует базу знаний

Проверьте:
- Инициализирован ли `self.knowledge_base` в DialogueManager
- Вызывается ли `kb.get_context_summary()` перед запросом
- Логи в `logs/dialogue_manager.log`

### Галлюцинации не детектируются

- Обновите паттерны в `should_verify_before_claiming()`
- Добавьте больше ключевых слов для детекции утверждений
