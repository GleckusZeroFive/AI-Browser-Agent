# SupervisorAgent Architecture

## Обзор

**SupervisorAgent** - это реактивный "старший брат" агента, который:
- ⚡ **НЕ тратит токены постоянно** (активируется только при ошибках)
- 🔍 Анализирует runtime errors с помощью LLM
- 📊 Фиксирует все ошибки для последующего анализа
- 💬 Общается с пользователем в production mode
- 🤖 Автоматически продолжает в sandbox mode

## Ключевая идея

### ❌ НЕ делаем (дорого):
```python
# Постоянное наблюдение = постоянные LLM вызовы = 💸💸💸
while True:
    result = executor.execute(action)
    supervisor.analyze(result)  # ← LLM каждый раз!
```

### ✅ Делаем (умно):
```python
# Реактивное наблюдение = LLM только при ошибках = 💰
try:
    result = await executor.execute(action)
except RuntimeError as e:
    supervisor.handle_runtime_error(e)  # ← LLM только здесь!
```

## Типы обработки ошибок

### 1. Structured Error (легковесная)
```python
# ActionExecutor возвращает structured error
result = {
    "status": "error",
    "error_type": "parameter_error",
    "message": "Элемент не найден",
    "suggestion": "Проверьте селектор"
}

# SupervisorAgent обрабатывает БЕЗ LLM:
# - Логирует в файл
# - Показывает пользователю
# Токены: 0 ✅
```

### 2. Runtime Error (тяжёлая)
```python
# Неожиданное исключение
try:
    result = await executor.execute(action)
except RuntimeError as e:
    # SupervisorAgent обрабатывает С LLM:
    # - Анализирует контекст (~500 токенов)
    # - Генерирует понятное сообщение
    # - Логирует с анализом
    # - Спрашивает пользователя
    # Токены: ~500 (разово) ✅
```

## Экономия токенов

### Типичная сессия (10 действий):

**БЕЗ Supervisor:**
```
10 действий × 0 токенов = 0 токенов
Но при ошибке → КРАШ 💥
```

**С Reactive SupervisorAgent:**
```
10 действий:
  - 9 успешных → 0 токенов (supervisor спит)
  - 1 structured error → 0 токенов (есть описание)

ИТОГО: 0 токенов, graceful degradation ✅

Если RuntimeError:
  - 1 LLM вызов ≈ 500 токенов (разово)

ИТОГО: 500 токенов, агент НЕ падает + анализ ✅
```

### Сравнение с постоянным наблюдением:

| Подход | Токены на 10 действий | Результат |
|--------|----------------------|-----------|
| **Без Supervisor** | 0 | ❌ Краш при ошибке |
| **Proactive Supervisor** | 10 × 500 = 5000 | ✅ Нет краша, но дорого 💸 |
| **Reactive Supervisor** | 0-500 (только если ошибка) | ✅✅ Нет краша + дёшево 🚀 |

## Режимы работы SupervisorAgent

| Аспект | Production Mode | Sandbox Mode |
|--------|----------------|--------------|
| **Перехват ошибок** | ✅ Да | ✅ Да |
| **LLM анализ** | ✅ Только runtime errors | ✅ Все ошибки |
| **Логирование** | ✅ session_*.json | ✅ sandbox_detail_*.json |
| **Взаимодействие с пользователем** | ✅ Спрашивает контекст | ❌ Автоматически |
| **Попытки восстановления** | ✅ Показывает suggestion | ✅ Тестирует alternatives |
| **Отчёт** | Краткий (errors summary) | Детальный (с рекомендациями) |

## Преимущества

1. **Экономия токенов** - LLM вызовы только при ошибках (редко)
2. **Graceful degradation** - агент не падает, а объясняет проблему
3. **Обучение на ошибках** - sandbox генерирует тесты из реальных кейсов
4. **Автоматический анализ** - не нужно вручную воспроизводить баги
5. **Поиск решений** - sandbox ищет работающие альтернативы

## Использование

### В коде

```python
from src.agent.supervisor_agent import SupervisorAgent

# Создаём supervisor (один раз)
supervisor = SupervisorAgent(mode="production")

# Оборачиваем все вызовы executor
result = await supervisor.supervised_execute(executor, action)

# При завершении - сохраняем статистику
supervisor.save_session_summary()
```

```bash
# Обычная работа (с автоматической фиксацией ошибок)
```

## FAQ

**Q: Сколько стоят токены?**
A: В типичной сессии: 0-500 токенов (только если runtime error). Это ~$0.001-0.002.

**Q: Sandbox всегда нужен?**
A: Нет! Sandbox запускается только когда хочешь проанализировать накопленные ошибки.

**Q: Что если нет ошибок?**
A: Sandbox выведет сообщение "Нет ошибок для анализа" и завершится.

**Q: Можно ли отключить Supervisor?**
A: Технически да, но не рекомендуется. Он не тратит токены пока всё работает.

**Q: Supervisor заменяет обычные тесты?**
A: Нет, это дополнение. Он анализирует РЕАЛЬНЫЕ ошибки из production, которые не покрыты тестами.

## Заключение

SupervisorAgent превращает процесс отладки из реактивного в проактивный, при этом:
- 💰 Не тратит токены впустую
- 🚀 Не ломает пользовательский опыт
- 🧪 Автоматически учится на ошибках
- 🔍 Ищет решения без вмешательства разработчика
