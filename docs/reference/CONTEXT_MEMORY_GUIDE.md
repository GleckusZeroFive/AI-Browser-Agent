# Руководство: Как работает память контекста агента

## Концепция

Агент теперь различает **два типа памяти**:

### 1. Долговременная память (База знаний)
- Факты о пользователе: город, предпочтения, аллергии
- Проверенные данные: существующие рестораны, сервисы
- Сохраняется между сессиями

### 2. Краткосрочная память (Контекст разговора)
- Что агент только что нашёл и показал
- Где агент находится (на какой странице)
- Какие действия агент уже выполнил
- Сохраняется в `conversation_history` текущей сессии

## Ключевая логика: "ПОКАЗАЛ → ВЫБРАЛ → КЛИКНУЛ"

```
┌─────────────────────────────────────────────┐
│  ЭТАП 1: АГЕНТ ИЩЕТ И ПОКАЗЫВАЕТ           │
├─────────────────────────────────────────────┤
│  User: "Хочу пепперони"                     │
│  Agent:                                     │
│    1. navigate(delivery-club.ru)           │
│    2. search_and_type("пепперони")         │
│    3. get_page_text() → анализирует        │
│    4. Показывает: "Нашёл 3 варианта..."    │
│                                             │
│  Память: [На Delivery Club, показал 3]     │
└─────────────────────────────────────────────┘
         ↓
┌─────────────────────────────────────────────┐
│  ЭТАП 2: ПОЛЬЗОВАТЕЛЬ ВЫБИРАЕТ             │
├─────────────────────────────────────────────┤
│  User: "Пепперони премиум"                  │
│                                             │
│  Проверка агента:                           │
│  ❓ Это выбор из моего списка?              │
│     → ДА! (есть в показанных вариантах)    │
│  ❓ Я уже на нужной странице?               │
│     → ДА! (Delivery Club открыт)           │
│                                             │
│  Решение: КЛИКНУТЬ, не искать!             │
└─────────────────────────────────────────────┘
         ↓
┌─────────────────────────────────────────────┐
│  ЭТАП 3: АГЕНТ КЛИКАЕТ                     │
├─────────────────────────────────────────────┤
│  Agent:                                     │
│    click_by_text("Пепперони премиум")      │
│                                             │
│  ✅ Один клик вместо 10+ действий!         │
└─────────────────────────────────────────────┘
```

## Как агент отличает выбор от нового запроса

### Выбор из списка (не искать заново):
- "Пепперони премиум" ← название из списка
- "Третий вариант" ← ссылка на список
- "От Симона" ← часть названия из списка
- "Хочу подороже" ← свойство из показанных

### Новый запрос (искать заново):
- "Хочу бургер" ← другое блюдо
- "Найди в другом ресторане" ← явное требование
- "Давай попробуем Додо Пиццу" ← другая платформа

## Архитектура в коде

### AIAgent.conversation_history
```python
[
  {"role": "system", "content": "Ты агент..."},
  {"role": "user", "content": "Хочу пепперони"},
  {"role": "assistant", "content": "Нашёл 3 варианта:\n1. Пепперони..."},
  {"role": "user", "content": "Пепперони премиум"},  # ← Контекст сохранён!
  {"role": "assistant", "content": "{\"action\": \"click_by_text\"..."}
]
```

LLM видит **весь контекст** и понимает:
- Предыдущее сообщение показало 3 варианта
- "Пепперони премиум" есть в списке
- Значит нужно кликнуть, а не искать

## Промпт-инженеринг

### В ShoppingAgent промпте добавлено:

```text
⚡ ГЛАВНОЕ ПРАВИЛО: Запоминай что ты уже нашёл!

Если ты показал пользователю список вариантов:
✅ ТЫ УЖЕ НА ПРАВИЛЬНОЙ СТРАНИЦЕ
✅ ВАРИАНТЫ УЖЕ НАЙДЕНЫ И ВИДНЫ
✅ НЕ ИДИ ИСКАТЬ СНОВА!

Перед каждым действием спрашивай себя:
Q: Я только что показал варианты пользователю?
   → ДА: остаюсь на этой странице

Q: Пользователь выбирает из моего списка?
   → ДА: click_by_text по названию из списка
   → НЕТ: новый поиск
```

### Примеры в промпте

Промпт содержит **подробные примеры**:
- ✅ Правильное поведение с пояснением
- ❌ Неправильное поведение с объяснением ошибки
- Алгоритм проверки перед действием

## Практические советы

### Для разработчиков

1. **Тестируйте сценарий "показ → выбор":**
   ```bash
   python main.py
   # "Хочу пепперони"
   # [агент показывает список]
   # "Пепперони премиум"
   # [агент должен кликнуть, не искать!]
   ```

2. **Проверяйте conversation_history:**
   - Контекст сохраняется в списке сообщений
   - LLM видит все предыдущие ответы
   - Промпт напоминает использовать контекст

3. **Логируйте решения агента:**
   ```python
   self.logger.info(f"Reasoning: {action['reasoning']}")
   ```

   В `reasoning` должно быть:
   - ✅ "Кликаю на вариант #3 из показанного списка"
   - ❌ "Ищу пепперони премиум" (если уже показал!)

### Для улучшения промпта

Если агент продолжает искать заново:

1. **Усильте правила:**
   ```text
   🚨 КРИТИЧНО: НИКОГДА не начинай поиск заново
   если ты только что показал варианты пользователю!
   ```

2. **Добавьте больше примеров:**
   - Конкретные ошибочные сценарии
   - Правильные альтернативы

3. **Проверьте temperature:**
   - Высокий temperature = больше креативности = больше ошибок
   - Для следования правилам: `temperature=0.3-0.5`

## Отладка проблем

### Проблема: Агент ищет заново
**Диагностика:**
```python
# В dialogue_manager.py добавьте:
print(f"Last assistant message: {agent.conversation_history[-2]}")
print(f"User selected: {user_input}")
```

**Решение:**
- Проверьте что контекст в conversation_history
- Усильте инструкции в промпте
- Добавьте explicit проверку в коде

### Проблема: Агент не находит текст для клика
**Диагностика:**
```python
# В browser_controller.py:
print(f"Trying to click: {text}")
print(f"Available text on page: {page_text[:500]}")
```

**Решение:**
- Используйте частичное совпадение при click_by_text
- Добавьте fallback на нечёткий поиск

## Будущие улучшения

### 1. Явное хранение результатов
```python
class AIAgent:
    def __init__(self):
        self.last_shown_results = []  # Список показанных вариантов

    def is_selection(self, user_input: str) -> bool:
        """Проверяет является ли запрос выбором из списка"""
        return any(
            result.lower() in user_input.lower()
            for result in self.last_shown_results
        )
```

### 2. Контекстный граф
```python
class ContextGraph:
    """
    Граф состояний диалога:
    SEARCHING → SHOWING_RESULTS → SELECTED → DETAILS
    """

    def should_search_again(self, state, user_input) -> bool:
        if state == "SHOWING_RESULTS":
            return not self.is_selection(user_input)
        return True
```
