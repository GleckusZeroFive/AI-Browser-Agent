# Улучшения агента - Исправление проблем с памятью и поиском

## Обнаруженные проблемы

### 1. Агент не запоминает выбор пользователя
**Проблема:** После показа списка вариантов (например, "Пепперони премиум от Симона") агент начинает поиск заново вместо клика по уже найденному элементу.

**Пример:**
```
Agent: Нашёл несколько вариантов:
1. Пепперони — 960₽
2. Пепперони премиум — 1 320₽

User: "Пепперони премиум"
Agent: [начинает новый поиск вместо клика] ❌
```

### 2. Неправильная работа с URL Яндекс.Еды
**Проблема:** Агент использует неправильные URL: `yandex.ru/eats` или `yandex.ru/eda` вместо корректного `eda.yandex.ru/[город]?shippingType=delivery`

### 3. Путаница с контекстом
**Проблема:** Агент переспрашивает информацию, которую уже знает (город, предпочтения), и не помнит что он уже находится на странице с результатами.

## Внесённые изменения

**Файл:** `src/agent/specialized_agents.py` (строки 267-371)

- Проверка перед действием: "Я только что показал варианты?"
- Различие между выбором из списка и новым запросом
- Примеры правильного и неправильного поведения

- НЕ используй yandex.ru/eda - это старый URL!

### 4. Улучшена секция "БАЗА ЗНАНИЙ И ПАМЯТЬ ДИАЛОГА"
**Файл:** `src/agent/ai_agent.py` (строки 48-92)

Добавлено разделение на два типа памяти:
1. **Долговременная память** - факты о пользователе
2. **Краткосрочная память** - контекст текущего разговора

С явными правилами:
- ✅ Запоминай что ты уже нашёл

## Тестирование

Для проверки исправлений протестируйте следующий сценарий:

1. Запустите агента: `python main.py`
2. Введите: "Хочу пепперони" (или любое другое блюдо)
3. Дождитесь когда агент покажет список вариантов
4. Выберите один из показанных вариантов по названию
5. **Ожидание:** Агент должен кликнуть по выбранному варианту БЕЗ нового поиска
