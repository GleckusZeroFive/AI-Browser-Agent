Context Extraction Pattern - Architecture Diagram
================================================

┌─────────────────────────────────────────────────────────────────────┐
│                          DIALOGUE MANAGER                           │
│                       (Orchestrates the flow)                       │
└──────────────────┬──────────────────────────────────────────────────┘
                   │
                   │ 1. User Input
                   ├──────────────────────────────────────────────┐
                   │                                              │
                   ▼                                              │
         ┌─────────────────┐                                     │
         │    AI Agent     │                                     │
         │  (Responds to   │                                     │
         │   user input)   │                                     │
         └────────┬────────┘                                     │
                  │ 2. Response                                  │
                  │                                              │
                  ▼                                              │
    ┌──────────────────────────────┐                            │
    │    CONTEXT EXTRACTOR          │                            │
    │  (Automatic extraction)       │                            │
    │                               │                            │
    │  extract_from_turn()          │◄───────────────────────────┘
    │    ├─> LLM Analysis           │   3. Extract context from
    │    ├─> JSON Parsing           │      (user_msg, agent_response)
    │    └─> Update Context         │
    │                               │
    │  extracted_context: {         │
    │    "dietary_restrictions": ...,│
    │    "people_count": ...,       │
    │    "budget": ...,             │
    │  }                            │
    └──────────────┬────────────────┘
                   │
                   │ 4. Action Execution
                   │
                   ▼
         ┌─────────────────────────────────┐
         │   _execute_action_with_followup │
         │                                 │
         │   ┌───────────────────────────┐ │
         │   │ _format_action_result     │ │
         │   │                           │ │
         │   │ get_context_for_action()  │ │◄── 5. Get ONLY relevant
         │   │    ↓                      │ │      context for this
         │   │ Returns:                  │ │      specific action
         │   │ "⚠️ ВАЖНАЯ ИНФОРМАЦИЯ:"  │ │
         │   │ "🚨 ОГРАНИЧЕНИЯ: ..."    │ │
         │   └───────────────────────────┘ │
         └─────────────────────────────────┘

OLD vs NEW APPROACH
===================

OLD APPROACH ❌
---------------
┌──────────────────────────────────┐
│  DialogueManager                 │
│                                  │
│  user_restrictions = {           │ ← Not its responsibility!
│    "allergies": [],              │
│    "dislikes": [],               │
│    "preferences": []             │
│  }                               │
│                                  │
│  ┌────────────────────────────┐  │
│  │ _format_action_result      │  │
│  │                            │  │
│  │ context +=                 │  │
│  │   _format_user_restrictions│◄─┼─ Passed on EVERY action!
│  │                            │  │   (even for scroll!)
│  └────────────────────────────┘  │
└──────────────────────────────────┘

NEW APPROACH ✅
---------------
┌──────────────────────────────────┐
│  DialogueManager                 │
│                                  │
│  context_extractor = None        │ ← Delegated!
│                                  │
│  ┌────────────────────────────┐  │
│  │ _format_action_result      │  │
│  │                            │  │
│  │ context +=                 │  │
│  │   context_extractor        │◄─┼─ Only relevant context!
│  │   .get_context_for_action()│  │   Smart filtering!
│  └────────────────────────────┘  │
└──────────────────────────────────┘
            │
            ▼
┌──────────────────────────────────┐
│  ContextExtractor                │
│  (Separate responsibility)       │
│                                  │
│  • Automatic extraction          │
│  • Smart relevance filtering     │
│  • Context management            │
└──────────────────────────────────┘


BENEFITS
========

✅ AUTOMATIC
  LLM extracts context without explicit questions
  User: "Пицца без мяса" → Extracted: {"dietary_restrictions": "без мяса"}

✅ EFFICIENT
  Context is only passed when relevant
  scroll → no context
  click_by_text → dietary restrictions included

✅ CLEAN ARCHITECTURE
  Separate class with single responsibility
  DialogueManager → orchestrates
  ContextExtractor → manages context

✅ EXTENSIBLE
  Add new context type in 2 places:
  1. CONTEXT_RELEVANCE mapping
  2. Formatter function
