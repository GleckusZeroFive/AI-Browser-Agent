## [Unreleased] - 2026-01-15

### Добавлено

#### 🐛 Bug Lifecycle Management System - НОВОЕ!
- **Автоматическое отслеживание lifecycle багов** от обнаружения до закрытия
- **Статусы багов**: DETECTED → ANALYZED → FIXED → VERIFIED → CLOSED
- **BugTracker**: система управления багами с deduplicate повторений
- **Bug Manager CLI**: утилита командной строки для управления багами
- **Интеграция с SupervisorAgent**: автоматическая регистрация ошибок
- **Связь с git commits**: каждое исправление связано с commit hash
- **Автоматическая группировка**: одинаковые ошибки получают один bug_id
- Новые файлы:
  - `src/utils/bug_tracker.py` - система отслеживания lifecycle
  - `bug_manager.py` - CLI утилита
  - `BUG_LIFECYCLE_MANAGEMENT.md` - полная документация
  - `data/bugs/` - структурированное хранилище багов

**Решает критическую проблему**: Теперь логи отображают актуальное состояние, а не исторические ошибки. Видно какие баги активны, какие исправлены, какие верифицированы.

## [Previous] - 2026-01-15

### Добавлено

#### 📚 База знаний агента (Knowledge Base) - НОВОЕ!
- **Долговременная память агента** - сохранение информации между сеансами
- **Автоматическое извлечение** информации о пользователе (город, предпочтения, аллергии)
- **Проверенные факты** - база данных о существующих ресторанах, сервисах
- **История взаимодействий** - запоминание что пользователь одобрил/отклонил
- **Детектор галлюцинаций** - проверка непроверенных утверждений агента
- **Предупреждения пользователю** когда агент делает утверждения без проверки
- Новые файлы:
  - `src/agent/knowledge_base.py` - основной модуль базы знаний
  - `data/knowledge_base.json` - хранилище (создается автоматически)
  - `KNOWLEDGE_BASE.md` - подробная документация

#### 🔄 Автоматическая ротация и очистка логов - НОВОЕ!
- **Ограничение размера логов** - максимум 5 MB на файл
- **Автоочистка** - удаление логов старше 7 дней
- **Ротация файлов** - хранение 3 последних версий каждого лога
- **Эффективное хранение** - вместо сотен MB теперь ~15-20 MB максимум
- Новый файл: `src/utils/log_setup.py` - система управления логами

## Использование

### Запуск приложения с новыми проверками

```bash
python main.py
```

При запуске вы увидите:
1. Информацию о версиях установленных библиотек
2. Результаты проверки системных требований
3. Детальные инструкции если что-то не настроено

### Запуск тестов

```bash
# Установить dev зависимости
pip install -r requirements-dev.txt

# Запустить все тесты
pytest

# Запустить только unit тесты
pytest -m unit

# Запустить с покрытием
pytest --cov=src --cov-report=html
```

## Migration Guide

Для обновления до новой версии:

1. Установите обновленные зависимости:
   ```bash
   pip install -r requirements.txt
   ```

2. (Опционально) Установите dev зависимости для тестирования:
   ```bash
   pip install -r requirements-dev.txt
   ```

3. Убедитесь что Firefox установлен:
   ```bash
   playwright install firefox
   ```

4. Запустите приложение - системная проверка запустится автоматически:
   ```bash
   python main.py
   ```
