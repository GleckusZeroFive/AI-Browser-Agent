# Changelog - Keepalive и Timeout система

**Дата:** 2026-01-14

---

## 🎯 Цель

Решить проблему падения агента при длительном AFK пользователя, соблюдая баланс между удобством и оптимизацией ресурсов.

---

## ✨ Что добавлено

### 1. Таймаут на ввод пользователя
- ⏱️ **5 минут** ожидания → предупреждение "Вы ещё здесь?"
- ⏱️ **+1 минута** grace period → завершение сессии
- Асинхронная реализация без полной блокировки event loop

### 2. Keepalive проверка браузера
- 🔄 Каждые **60 секунд** проверяется состояние браузера
- 🔄 **Автоматическое переподключение** при падении
- Работает параллельно с ожиданием ввода

### 3. Retry логика для действий
- 🔁 Автоматический retry при ошибках браузера
- 🔁 Переподключение API при разрыве соединения
- Graceful fallback с уведомлениями

### 4. Структурированное логирование
- 📝 `logs/dialogue_manager.log` - полная история событий
- 📝 `logs/errors.log` - только ошибки с traceback
- 📝 `logs/agent_responses.log` - история ответов агента (как раньше)

### 5. Настраиваемые параметры
- ⚙️ Все таймауты настраиваются через [src/config.py](src/config.py)
- ⚙️ Можно отключить keepalive (не рекомендуется)

## 🎯 Результат

### До:
```
❌ Браузер падает при AFK → программа ломается
❌ API отваливается → молчаливая ошибка
❌ Нет логов → непонятно что произошло
❌ Браузер работает вечно → ест ресурсы
```

### После:
```
✅ Браузер падает → автоматическое переподключение
✅ API отваливается → retry с уведомлением
✅ Все события в логах → полная прозрачность
✅ Graceful shutdown через 6 минут AFK → экономия ресурсов
```
