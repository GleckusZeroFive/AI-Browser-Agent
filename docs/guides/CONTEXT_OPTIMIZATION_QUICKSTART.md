# 🚀 Быстрый старт: Оптимизация контекста

## Что это?

Автоматическая система экономии токенов для AI Browser Agent, которая:
- ✅ Снижает rate limit ошибки на 80-90%
- ✅ Экономит 50-70% токенов
- ✅ Автоматически адаптируется к размеру разговора

## Как использовать?

### 1. Включить DEBUG режим (опционально)

В файле `.env`:

```env
DEBUG_MODE=true
```

### 2. Запустить агента как обычно

```bash
python main.py
```

### 3. Использовать агента

```
Вы: Хочу заказать пиццу
Агент: [работает с оптимизированным контекстом]
```

После каждого запроса (если DEBUG_MODE=true) увидишь:

```
📊 СТАТИСТИКА ИСПОЛЬЗОВАНИЯ ТОКЕНОВ
  📋 Уровень контекста: COMPACT
  💰 Экономия KB: 62.5%
```

## Как это работает?

### Автоматический выбор уровня

```
Начало разговора    → FULL (максимум информации)
Средний разговор    → COMPACT (баланс)
Длинный разговор    → MINIMAL (экономия)
```

### Локализация по задачам

```python
# Shopping - только предпочтения в еде
agent.set_task_type("shopping")
# Показывает: "пицца, итальянская кухня"
# Скрывает: "Python, Docker" (это для job search)

# Job Search - только навыки
agent.set_task_type("job_search")
# Показывает: "Python, Docker, React"
# Скрывает: "пицца" (это для shopping)
```

## Уровни детализации

| Уровень | Токенов | Когда используется |
|---------|---------|-------------------|
| MINIMAL | ~300 | Длинная история, экономия токенов |
| COMPACT | ~800 | Нормальный режим (по умолчанию) |
| FULL | ~4000 | Начало разговора, много места |

## FAQ

**Q: Влияет ли это на качество ответов?**
A: Нет. MINIMAL содержит все критичные данные (город, аллергии). Качество сохраняется.

**Q: Нужно ли что-то настраивать?**
A: Нет. Всё работает автоматически. DEBUG_MODE нужен только для мониторинга.

**Q: Что если нужен всегда FULL контекст?**
A: В `src/config.py` увеличь `SAFE_TOKEN_MARGIN = 0.9` (90% вместо 70%).

**Q: Как часто переключается уровень?**
A: При каждом запросе система пересчитывает доступное место и выбирает оптимальный уровень.

**Q: Работает ли с всеми моделями?**
A: Да. Система знает лимиты всех моделей из `Config.MODEL_TOKEN_LIMITS`.

## Troubleshooting

### Постоянно MINIMAL
→ История слишком большая. Сократи: `agent._trim_conversation_history(5000)`

### Язык английский
→ Все промпты содержат "ОТВЕЧАЙ НА РУССКОМ ЯЗЫКЕ". Проверь что используешь `specialized_agent`.

### Rate limit всё равно
→ Увеличь задержку: `Config.MIN_REQUEST_INTERVAL = 2.0` в `src/config.py`
