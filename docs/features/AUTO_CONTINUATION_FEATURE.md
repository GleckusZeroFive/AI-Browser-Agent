# Автоматическое продолжение поиска при достижении лимита действий

**Дата:** 2026-01-14
**Проблема:** Агент молчит после выполнения действий, требует подталкивания от пользователя
**Статус:** ✅ Реализовано

---

## 🐛 Проблема

### Описание исходной ситуации

Пользователь сообщил:
> "Агент не выдал результата, он сообщил о том, что ничего не нашёл. У пользователя может сложиться фрустрация от того, что агента надо постоянно подталкивать."

**Пример из реальной сессии:**
```
👤 Пользователь: Нас 4 человека, любим острые мексиканские блюда
🤖 Агент: [выполняет 7 действий]
🤖 Агент: "Я ищу блюда мексиканской кухни, но на текущей странице таких блюд не нашлось. Я продолжаю прокручивать страницу."
[МОЛЧАНИЕ]
👤 Пользователь: А дальше?
```

### Техническая причина

1. **Лимит действий слишком мал**: 10 действий может не хватить для сложных запросов
2. **Агент выдаёт технические статусы**: "Я ищу...", "Я прокручиваю..." вместо результатов
3. **Нет автоматического продолжения**: При достижении лимита агент просто останавливается

**Анализ логов (18:42-18:43):**
```
18:42:35 - navigate
18:42:53 - get_page_text
18:42:57 - find_text
18:43:02 - scroll_down
18:43:06 - scroll_down
18:43:10 - scroll_down
18:43:14 - scroll_down
18:43:21 - scroll_down
18:43:26 - find_text
18:43:31 - [HTTP Request]
18:43:40 - Завершение по команде пользователя
```

Агент выполнил только 7 действий и остановился, хотя мог продолжить (лимит 10).

---

## ✅ Решение

### Концепция

**Автоматическое продолжение** - когда агент достигает лимита действий но НЕ дал финального результата:

1. Система определяет что агент не нашёл результат
2. Уведомляет пользователя: "Продолжаю поиск, требуется чуть больше времени..."
3. Сбрасывает счётчик действий и начинает новый цикл
4. Агент продолжает поиск с сохранённым контекстом

**Лимиты:**
- 10 действий в одном цикле
- 3 цикла максимум (итого 30 действий)

### ВАЖНО: Исправление после первого теста

**Дата:** 2026-01-14 (вечер)

**Проблема:** При первом тесте агент выполнил только 5 действий и остановился БЕЗ вывода результата. Автопродолжение не сработало, т.к. проверка `_should_continue_search` выполнялась только при `actions_count >= 10`.

**Исправление:** Теперь `_should_continue_search` вызывается **ВСЕГДА** когда агент не дал следующего действия, независимо от количества выполненных действий. Если нужно продолжить:
- При `actions_count >= 10` → новый цикл
- При `actions_count < 10` → продолжение в том же цикле

### 1. Метод определения необходимости продолжения

**Файл:** [src/dialogue_manager.py](src/dialogue_manager.py) строки 386-441

```python
def _should_continue_search(self, response: str) -> bool:
    """
    Определить, нужно ли агенту продолжать поиск

    Критерии для продолжения:
    - Ответ пустой или очень короткий
    - Ответ содержит JSON (технический ответ)
    - Ответ содержит промежуточные статусы ("ищу", "продолжаю", "не нашёл")

    Критерии для завершения:
    - Ответ содержит конкретную рекомендацию или результат
    - Ответ содержит цены, названия блюд
    - Ответ содержит финальный негативный результат
    """
    response = response.strip()

    # Пустой или очень короткий - продолжаем
    if not response or len(response) < 20:
        return True

    # JSON ответ (технический) - продолжаем
    if response.startswith("{") and response.endswith("}"):
        return True

    response_lower = response.lower()

    # Промежуточные фразы - продолжаем
    intermediate_phrases = [
        "продолжаю", "ищу", "прокручиваю", "не нашёл",
        "не нашлось", "пока не вижу", "еще ищу"
    ]
    if any(phrase in response_lower for phrase in intermediate_phrases):
        return True

    # Финальные фразы - завершаем
    final_phrases = [
        "рекомендую", "предлагаю", "нашёл", "есть вариант",
        "подойдёт", "к сожалению", "увы", "не могу найти"
    ]
    if any(phrase in response_lower for phrase in final_phrases):
        return False

    # Если содержит цену - это конкретный результат
    if "₽" in response or "руб" in response_lower:
        return False

    # По умолчанию - завершаем
    return False
```

### 2. Логика автоматического продолжения

**Файл:** [src/dialogue_manager.py](src/dialogue_manager.py) строки 141-300

**Ключевые изменения:**

```python
# Было (старая версия):
max_actions = 10
actions_count = 0

while current_action and actions_count < max_actions:
    # ...выполнение действий...

# Стало (новая версия):
max_actions_per_cycle = 10  # Действий в одном цикле
max_cycles = 3  # Максимум циклов (итого 30 действий)

actions_count = 0  # Счётчик в текущем цикле
cycle_count = 0  # Номер цикла
total_actions = 0  # Всего действий

while current_action and cycle_count < max_cycles:
    # ...выполнение действий...

    # Когда агент не дал следующего действия
    if not next_action:
        # ВСЕГДА проверяем нужно ли автопродолжение
        should_continue = self._should_continue_search(response)

        if should_continue and cycle_count < max_cycles - 1:
            # Агент НЕ дал финального результата

            if actions_count >= max_actions_per_cycle:
                # Достигнут лимит цикла - начинаем новый цикл
                cycle_count += 1
                actions_count = 0
                print(f"\n⏳ Продолжаю поиск (цикл {cycle_count + 1}/{max_cycles})...")
            else:
                # Лимит не достигнут, но агент не дал результата - продолжаем
                print(f"\n⏳ Продолжаю поиск...")

            # Контекст продолжения для агента
            continuation_prompt = f"""Выполнено {total_actions} действий.
Промежуточный результат: {response[:200] if response else '(нет ответа)'}...
Продолжай поиск. У тебя есть ещё {max_actions_per_cycle - actions_count} действий."""

            continuation_response = self.agent.chat(continuation_prompt)
            current_action = self.agent.parse_action(continuation_response)
```

**Важное отличие:** Проверка `_should_continue_search` теперь выполняется **независимо от количества действий**. Если агент выполнил 5 действий и не дал результата - автопродолжение сработает.
