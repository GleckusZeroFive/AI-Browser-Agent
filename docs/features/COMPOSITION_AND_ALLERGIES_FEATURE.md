# Улучшение работы с составом блюд и проверка аллергий

**Дата:** 2026-01-14
**Проблема:** Агент галлюцинирует несуществующие блюда, не учитывает состав и аллергии
**Статус:** ✅ Реализовано

---

## 🐛 Проблема

### Описание исходной ситуации

Пользователь протестировал агента с запросом: "Мой масик и я хотим что-нибудь лакшери. С креветкой, да побогаче."

**Что пошло не так:**
```
19:04:51 - Выполняю действие: click_by_text
19:05:01 - ERROR: Ошибка клика по тексту: Timeout 10000ms exceeded.
         waiting for get_by_text("Креветочная пицца").first

19:05:36 - Выполняю действие: click_by_text (повторная попытка)
19:05:46 - ERROR: Timeout... "Креветочная пицца"

19:07:17 - ERROR: Timeout... "Креветочная пицца" (3-я попытка)
19:07:45 - ERROR: Timeout... "Креветочная пицца" (4-я попытка)
```

Агент **потратил 21 действие** пытаясь кликнуть на **несуществующее блюдо** "Креветочная пицца".

### Технические причины

1. **Галлюцинации названий**: Агент придумывает несуществующие блюда вместо анализа реального меню
2. **Игнорирование ошибок**: После ошибки клика агент пытается снова кликнуть на то же самое
3. **Фокус на названиях**: Агент ищет "Креветочная пицца" по названию, а не анализирует состав реальных блюд
4. **Нет проверки аллергий**: Агент не спрашивает об аллергиях проактивно

### Отзыв пользователя

> "Он настойчивый, это хорошо, но работает очень медленно. Задачу не выполнил, так как пытался кликнуть на несуществующее блюдо. Давай сделаем его больше ориентирующимся на состав блюд, а не на названия, он галлюцинирует."

> "Добавим ещё уточнения для аллергиков. Проверка на нежелательные ингредиенты обязательна, пользователь может забыть самостоятельно написать, есть ли у него неприязнь к помидорам или аллергия на морепродукты."

---

## ✅ Решение

### Концепция

**Три ключевых улучшения:**

1. **Запрет на галлюцинации**
   - Кликать ТОЛЬКО на то, что было в `get_page_text`
   - После ошибки клика - искать ДРУГОЕ блюдо
   - Не придумывать названия

2. **Фокус на составе, не на названиях**
   - "С креветками" → искать в составе слово "креветки"
   - "Острое" → искать "перец", "халапеньо", "чили" в составе
   - Читать состав в модальном окне (`get_modal_text`)

3. **Проактивная проверка аллергий**
   - ВСЕГДА спрашивать об аллергиях/ограничениях
   - Запоминать ограничения пользователя
   - Проверять состав перед рекомендацией

### 2. Система отслеживания ограничений

**Файл:** [src/dialogue_manager.py](src/dialogue_manager.py)

#### Хранилище ограничений (строки 25-30)

```python
# Отслеживание ограничений пользователя (аллергии, предпочтения)
self.user_restrictions = {
    "allergies": [],  # Список аллергенов ["морепродукты", "орехи"]
    "dislikes": [],   # Список нежелательных продуктов ["помидоры", "грибы"]
    "preferences": [] # Предпочтения ["острое", "без мяса"]
}
```

#### Метод форматирования ограничений (строки 346-365)

```python
def _format_user_restrictions(self) -> str:
    """Форматировать ограничения пользователя для контекста агента"""
    parts = []

    if self.user_restrictions["allergies"]:
        allergies = ", ".join(self.user_restrictions["allergies"])
        parts.append(f"🚨 АЛЛЕРГИИ: {allergies}")

    if self.user_restrictions["dislikes"]:
        dislikes = ", ".join(self.user_restrictions["dislikes"])
        parts.append(f"❌ НЕ ЕСТ: {dislikes}")

    if self.user_restrictions["preferences"]:
        prefs = ", ".join(self.user_restrictions["preferences"])
        parts.append(f"✅ ПРЕДПОЧИТАЕТ: {prefs}")

    if parts:
        return "\n⚠️ ОГРАНИЧЕНИЯ ПОЛЬЗОВАТЕЛЯ:\n" + "\n".join(parts)

    return ""
```

#### Интеграция в контекст действий (строки 339-342)

```python
# Добавляем ограничения пользователя в контекст
restrictions_text = self._format_user_restrictions()
if restrictions_text:
    context += f"\n{restrictions_text}"
```

Теперь при каждом действии агент видит ограничения пользователя в контексте!
