# AI Browser Agent - Додо Пицца

AI агент для автоматизации заказа еды через браузер с использованием искусственного интеллекта.

## Требования системы

### Операционные системы
- Linux (Ubuntu 20.04+, Debian 11+)
- macOS 11+
- Windows 10/11

### Программное обеспечение
- Python 3.10 или выше
- pip (менеджер пакетов Python)
- 2 GB свободного места на диске
- Стабильное интернет-соединение

### Дополнительно
- API ключ Groq (бесплатная регистрация на [groq.com](https://console.groq.com))

## Установка

### Шаг 1: Клонирование репозитория

```bash
git clone <repository-url>
cd ai-browser-agent
```

### Шаг 2: Создание виртуального окружения

```bash
# Linux/macOS
python3 -m venv venv
source venv/bin/activate

# Windows
python -m venv venv
venv\Scripts\activate
```

### Шаг 3: Установка зависимостей

```bash
pip install -r requirements.txt
```

### Шаг 4: Установка браузера Firefox для Playwright

```bash
playwright install firefox
```

**Важно:** Playwright автоматически загрузит и установит браузер Firefox. Эта команда необходима для корректной работы агента.

### Шаг 5: Настройка переменных окружения

Создайте файл `.env` в корневой директории проекта:

```bash
cp .env.example .env  # если есть .env.example
# или создайте файл вручную
```

Добавьте в файл `.env` ваш API ключ Groq:

```env
# API ключ для Groq
GROQ_API_KEY=your_groq_api_key_here

# Настройки Groq (опционально)
# GROQ_BASE_URL=https://api.groq.com/openai/v1
```

### Шаг 6: Проверка установки

```bash
python main.py
```

Если всё настроено правильно, вы увидите приветственное сообщение агента.

## 🚀 Оптимизация контекста (NEW!)

Система автоматически оптимизирует использование токенов для снижения rate limit ошибок и ускорения работы:

### Ключевые возможности

- **3 уровня детализации**: MINIMAL (~300т), COMPACT (~800т), FULL (~4000т)
- **Автоматический выбор**: система сама выбирает оптимальный уровень в зависимости от истории разговора
- **Экономия 50-70%**: значительное снижение использования токенов
- **Меньше rate limit**: снижение ошибок на 80-90%
- **Локализация по задачам**: контекст фильтруется по типу задачи (shopping/email/job_search)

### Как это работает

```
История пустая    → FULL (вся информация доступна)
История средняя   → COMPACT (баланс между деталями и экономией)
История полная    → MINIMAL (только критичная информация)
```

Система автоматически:
1. Отслеживает размер conversation history
2. Выбирает оптимальный уровень контекста для базы знаний
3. Подбирает соответствующий промпт для specialized agent
4. Сжимает историю при необходимости
5. Переключается между моделями при достижении лимитов

### Включение DEBUG режима

Для просмотра статистики использования токенов добавь в `.env`:

```env
# Режим отладки (показывает статистику токенов)

```

## Технические решения

### Выбор Groq вместо Claude/OpenAI

Проект использует **Groq** в качестве провайдера LLM вместо Claude (Anthropic) или OpenAI по следующим причинам:

#### Преимущества Groq

1. **Бесплатный доступ**
   - Groq предоставляет бесплатный tier с щедрыми лимитами
   - Claude и OpenAI требуют платную подписку для production использования
   - Идеально для тестирования и разработки

2. **Высокая скорость инференса**
   - Llama 4 Scout работает на скорости **>460 tokens/s** на специализированном hardware Groq
   - В 5-10 раз быстрее стандартных API
   - Критично для браузерной автоматизации, где важна быстрая реакция

3. **Доступные модели**

   Проект использует три модели в зависимости от задачи:

   | Модель | Назначение | Характеристики |
   |--------|-----------|----------------|
   | `meta-llama/llama-4-scout-17b-16e-instruct` | Быстрые действия (ShoppingAgent) | 17B активных параметров, низкий rate limit |
   | `llama-3.3-70b-versatile` | Сложный анализ (EmailAgent, JobSearchAgent) | 70B параметров, точный анализ текста |
   | `meta-llama/llama-4-maverick-17b-128e-instruct` | Fallback и рассуждения | 17B активных, 128 экспертов, 400B total |

4. **OpenAI-совместимый API**
   - Groq использует стандартный OpenAI API формат
   - Легко переключиться на OpenAI/Azure в production если потребуется
   - Используем библиотеку `openai` с изменённым `base_url`

5. **Rate Limit Management**
   - Проект реализует умную систему fallback между моделями
   - При достижении rate limit на одной модели автоматически переключается на другую
   - Минимизирует простои и обеспечивает стабильную работу

#### Сравнение с альтернативами

| Критерий | Groq | Claude (Anthropic) | OpenAI |
|----------|------|-------------------|--------|
| Бесплатный tier | ✅ Щедрый | ❌ Нет (только trial) | ⚠️ Ограниченный |
| Скорость | ✅ 460+ tok/s | ⚠️ Стандартная | ⚠️ Стандартная |
| Качество | ✅ Llama 4/3.3 | ✅ Claude Opus 4.5 | ✅ GPT-4 |
| Стоимость (paid) | ✅ $0.11/M tokens | ❌ Дорого | ⚠️ Средне |
| API совместимость | ✅ OpenAI format | ❌ Proprietary | ✅ Native |

### Sub-agent Architecture

Проект реализует архитектуру специализированных агентов (один из паттернов из требований):

- **EmailAgent** - эксперт по работе с почтой, определению спама
- **ShoppingAgent** - специалист по заказу еды и покупкам, учитывает аллергии
- **JobSearchAgent** - помощник по поиску работы и составлению откликов

Каждый агент имеет:
- Кастомный system prompt для своего домена
- Оптимальную модель (быстрая для простых действий, точная для анализа)
- Ключевые слова для автоматической активации

### Security Layer

Реализован security layer для предотвращения деструктивных действий:

- `_is_destructive_action()` - определяет опасные действия (оплата, удаление, отправка)
- `_ask_user_confirmation()` - запрашивает подтверждение перед выполнением
- Автоматическое распознавание финансовых операций, удаления данных, отправки форм
- Интеграция с DialogueManager для перехвата действий перед выполнением

### Обработка ошибок и Retry логика

- Автоматическое переключение между моделями при rate limit
- Восстановление браузерной сессии при падении
- Keepalive проверки состояния браузера
- Graceful degradation при недоступности API

### Конфигурация Demo Mode

Настройки находятся в файле [demo_config.yaml](demo_config.yaml):

```yaml
demo_mode:
  enabled: false  # Изменить на true для постоянного включения

  delays:
    before_action: 1.0    # Задержка перед действием (сек)
    after_action: 0.5     # Задержка после действия (сек)
    visual_indicator: 1.5 # Длительность визуальных индикаторов (сек)

  visual_markers:
    enabled: true           # Включить визуальные маркеры
    highlight_clicks: true  # Подсветка кликов
    show_typing: true       # Анимация ввода текста
    show_scroll: true       # Индикатор скролла

  logging:
    level: verbose          # Уровень детализации: minimal, normal, verbose
    show_code_line: true    # Показывать код перед выполнением
    show_duration: true     # Показывать время выполнения
```

## 🔐 Обработка капчи (CAPTCHA)

AI Browser Agent теперь включает встроенную систему детектирования и обработки капч!

### Автоматическое детектирование

Агент автоматически распознаёт популярные типы капч:
- ✅ Google reCAPTCHA v2/v3
- ✅ hCaptcha
- ✅ CloudFlare Turnstile
- ✅ Yandex SmartCaptcha

### Как это работает

1. **Автоматическое детектирование** - при переходе на страницу агент проверяет наличие капчи
2. **Пауза для решения** - если капча обнаружена, агент автоматически паузирует работу
3. **Уведомление пользователя** - вы получите понятное сообщение с инструкциями
4. **Ожидание решения** - агент ждёт, пока вы решите капчу вручную (до 5 минут)
5. **Автоматическое продолжение** - после решения работа возобновляется

### Пример работы

```
⚠️  Обнаружена Google reCAPTCHA v2
⏳ Пожалуйста, решите капчу вручную в браузере
⏱️  Ожидание: 300 секунд

[Вы решаете капчу в браузере]

✅ Капча решена за 23.4с
Продолжаем работу...
```

### Антидетект функции

Для минимизации появления капч агент использует:

- 🎭 **Стелс режим** - скрытие признаков автоматизации
- 🌐 **Реалистичный User-Agent** - ротация между настоящими браузерами
- 🖼️ **Canvas/WebGL защита** - защита от fingerprinting
- 📍 **Геолокация** - реалистичные координаты (Москва)
- 🕐 **Задержки между действиями** - имитация человеческого поведения

### Настройка

В файле `.env`:
```env
# Обработка капч (по умолчанию включена)
CAPTCHA_AUTO_HANDLE=true
CAPTCHA_MANUAL_TIMEOUT=300  # 5 минут на решение

# Опционально: Автоматическое решение (платные сервисы)
# CAPTCHA_SOLVER_SERVICE=2captcha
# CAPTCHA_SOLVER_API_KEY=your_api_key
```

## Troubleshooting (Решение проблем)

### Проблема: `playwright: command not found`

**Решение:**
Убедитесь, что виртуальное окружение активировано:
```bash
source venv/bin/activate  # Linux/macOS
venv\Scripts\activate     # Windows
```

Затем переустановите зависимости:
```bash
pip install -r requirements.txt
playwright install firefox
```

### Проблема: `Error: Browser was not installed`

**Решение:**
Выполните установку браузера Firefox:
```bash
playwright install firefox
```

Если проблема сохраняется, попробуйте установить все браузеры:
```bash
playwright install
```

### Проблема: `GROQ_API_KEY not found`

**Решение:**
1. Убедитесь, что файл `.env` существует в корневой директории проекта
2. Проверьте, что в файле `.env` указан ваш API ключ:
   ```env
   GROQ_API_KEY=your_actual_api_key_here
   ```
3. Получите API ключ на [console.groq.com](https://console.groq.com) если его нет

### Проблема: `Module not found` или `Import Error`

**Решение:**
Переустановите зависимости:
```bash
pip install --upgrade -r requirements.txt
```

Убедитесь, что используется правильная версия Python (3.10+):
```bash
python --version
```

### Проблема: Браузер не открывается

**Решение:**
1. Проверьте настройку `BROWSER_HEADLESS` в файле [src/config.py](src/config.py)
2. Для видимого браузера установите `BROWSER_HEADLESS = False`
3. Для фонового режима установите `BROWSER_HEADLESS = True`

### Проблема: Агент не может найти элементы на странице

**Решение:**
1. Проверьте интернет-соединение
2. Убедитесь, что сайт Додо Пицца доступен
3. Проверьте логи в папке `logs/` для детальной информации
4. Попробуйте перезапустить агента

### Проблема: Ошибки на Linux при установке Playwright

**Решение:**
Установите системные зависимости:

Ubuntu/Debian:
```bash
sudo apt-get update
sudo apt-get install -y \
    libnss3 \
    libnspr4 \
    libatk1.0-0 \
    libatk-bridge2.0-0 \
    libcups2 \
    libdrm2 \
    libdbus-1-3 \
    libxkbcommon0 \
    libxcomposite1 \
    libxdamage1 \
    libxfixes3 \
    libxrandr2 \
    libgbm1 \
    libpango-1.0-0 \
    libcairo2 \
    libasound2
```

Или используйте команду Playwright для автоматической установки:
```bash
playwright install-deps firefox
```

### Проблема: Высокое использование памяти

**Решение:**
1. Закройте другие приложения для освобождения памяти
2. Используйте headless режим (`BROWSER_HEADLESS = True` в [src/config.py](src/config.py))
3. Перезапустите агента

### Получение помощи

Если проблема не решена:
1. Проверьте логи в папке `logs/`
2. Посмотрите скриншоты в папке `screenshots/` для визуальной диагностики
3. Создайте issue в репозитории с описанием проблемы и логами
