# Удаление спама из почты

Этот пример демонстрирует использование AI Browser Agent для автоматизации работы с электронной почтой - удаление спама, очистка папок, фильтрация писем.

## Поддерживаемые почтовые сервисы

- **Яндекс.Почта** (mail.yandex.ru)
- **Gmail** (mail.google.com)

## Установка и настройка

### Требования

- Python 3.10+
- Установленный AI Browser Agent (см. основной [README.md](../../README.md))
- Учетная запись на Яндекс.Почте или Gmail

### Быстрый старт

```bash
# Из корневой директории проекта
python examples/email_spam/delete_spam.py
```

## Возможности

### Основные функции

1. **Удаление писем по отправителю**
   - Найти все письма от конкретного адреса
   - Массовое удаление спам-писем

2. **Очистка папок**
   - Папка "Спам"
   - Папка "Корзина"
   - Старые письма из "Входящих"

3. **Фильтрация по критериям**
   - Тема письма
   - Дата получения
   - Размер письма
   - Наличие вложений

4. **Массовые операции**
   - Удаление писем старше N дней
   - Перемещение в папку "Спам"
   - Безвозвратное удаление

## Примеры использования

### Базовые команды

#### Удаление писем от спамера

```
Вы: Открой Яндекс почту
Агент: Открываю mail.yandex.ru... Пожалуйста, войдите в свой аккаунт.
Вы: Удали все письма от spam@example.com
Агент: Ищу письма от spam@example.com... Найдено 15 писем. Удалить их?
Вы: Да
Агент: Удаляю письма... Готово! Удалено 15 писем.
```

#### Очистка папки спам

```
Вы: Зайди в Gmail и очисти папку Спам
Агент: Открываю Gmail... Пожалуйста, авторизуйтесь.
[После входа]
Агент: Перехожу в папку "Спам"... В папке 247 писем. Удалить все?
Вы: Да, удали все
Агент: Очищаю папку... Готово! Папка "Спам" очищена.
```

#### Удаление старых писем

```
Вы: Удали все письма старше 90 дней
Агент: Ищу письма старше 90 дней... Найдено 128 писем. Показать детали или сразу удалить?
Вы: Покажи детали
Агент:
  - Рассылки: 85 писем
  - Уведомления: 32 письма
  - Прочие: 11 писем
Вы: Удали только рассылки
Агент: Удаляю 85 писем... Готово!
```

## Безопасность

### Важные моменты

1. **Учетные данные**
   - Агент НЕ сохраняет ваши логины и пароли
   - Вход в почту выполняется вручную
   - Все данные остаются только в вашем браузере

2. **Подтверждение действий**
   - Агент всегда спрашивает подтверждение перед массовым удалением
   - Вы можете остановить процесс в любой момент

3. **Логи**
   - Пароли не записываются в логи
   - Сохраняются только действия агента (клики, переходы)

### Рекомендации

- Используйте двухфакторную аутентификацию на почте
- Регулярно проверяйте папку "Спам" перед массовым удалением
- Делайте бэкапы важных писем

## Частые проблемы

### Агент не может войти в почту

**Проблема:** Агент пытается ввести данные, но форма не заполняется.

**Решение:**
- Войдите в почту вручную
- Агент дождется авторизации и продолжит работу

### Не находит письма

**Проблема:** Агент говорит "писем не найдено", хотя они есть.

**Решение:**
- Уточните критерии поиска
- Проверьте правильность написания адреса/темы
- Попробуйте использовать поиск почтового сервиса

### Медленная работа

**Проблема:** Агент долго ищет и удаляет письма.

**Решение:**
- Это нормально для больших объемов
- Почтовые сервисы имеют ограничения на скорость операций
- Подождите или разделите задачу на части

### Капча или дополнительная проверка

**Проблема:** Почтовый сервис показывает капчу.

**Решение:**
- Пройдите проверку вручную
- После этого агент продолжит работу

## Технические детали

### Как это работает

1. **Запуск браузера**
   - Открывается Firefox с Playwright
   - Загружается почтовый сервис

2. **Навигация**
   - Агент анализирует структуру страницы
   - Находит нужные элементы (кнопки, письма, папки)

3. **Поиск писем**
   - Использует встроенный поиск почты
   - Применяет фильтры и критерии

4. **Выполнение действий**
   - Выделение писем
   - Клики по кнопкам удаления
   - Подтверждение операций

### Ограничения

- Работает только с веб-версией почты
- Не поддерживает IMAP/POP3
- Требует видимый браузер для ручного входа
- Зависит от структуры веб-интерфейса почты
