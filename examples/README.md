# Примеры использования AI Browser Agent

Эта директория содержит готовые примеры и сценарии использования AI Browser Agent для различных задач автоматизации браузера.

## Как использовать примеры

### Шаг 1: Установка

Убедитесь, что вы установили AI Browser Agent согласно инструкциям в [основном README](../README.md).

```bash
# Установка зависимостей
pip install -r requirements.txt

# Установка браузера
playwright install firefox

# Настройка .env файла
cp .env.example .env
# Добавьте ваш GROQ_API_KEY в .env
```

### Шаг 2: Запуск примера

```bash
# Из корневой директории проекта
python examples/<название_примера>/<скрипт>.py

# Например:
python examples/email_spam/delete_spam.py
python examples/job_search/search_hh.py
```

### Шаг 3: Взаимодействие

После запуска примера:

1. Дождитесь загрузки браузера
2. Следуйте инструкциям на экране
3. Давайте команды на естественном языке
4. Агент выполнит задачу автоматически

## Создание своего примера

Вы можете создать собственный пример использования агента. Вот базовый шаблон:

```python
"""
Пример: Описание вашего примера

Краткое описание того, что делает скрипт.
"""
import asyncio
import sys
from pathlib import Path

# Добавляем корневую директорию в путь
sys.path.insert(0, str(Path(__file__).parent.parent.parent))

from src.dialogue_manager import DialogueManager
from src.system_check import run_system_check
from src.version_info import print_version_info


async def main():
    """Запуск вашего примера"""
    print("=" * 70)
    print("🤖 AI BROWSER AGENT - ВАШ ПРИМЕР")
    print("=" * 70)

    # Проверка системы
    print_version_info(verbose=False)
    if not run_system_check():
        print("\n❌ Запуск невозможен.")
        sys.exit(1)

    # Запуск диалога
    manager = DialogueManager()

    # Добавьте специальный промпт для вашей задачи
    manager.agent.add_message("system", """
    Здесь опишите специальные инструкции для агента
    для выполнения вашей конкретной задачи.
    """)

    print("\n🤖 Агент готов к работе!")
    await manager.start()


if __name__ == "__main__":
    try:
        asyncio.run(main())
    except KeyboardInterrupt:
        print("\n\n👋 Программа завершена.")
    except Exception as e:
        print(f"\n\n❌ Критическая ошибка: {e}")
        sys.exit(1)
```

## Лучшие практики

### 1. Специализированные промпты

Для каждого примера создавайте специальный системный промпт, который:
- Описывает задачу агента
- Определяет правила поведения
- Указывает важные детали (безопасность, этика)
- Приводит примеры команд

### 2. Обработка ошибок

Всегда включайте:
- Try-catch блоки для обработки исключений
- Проверку системных требований
- Информативные сообщения об ошибках
- Логирование для отладки

### 3. Документация

Каждый пример должен иметь:
- README.md с описанием
- Примеры команд
- Раздел troubleshooting
- Информацию о безопасности

### 4. Безопасность

При работе с чувствительными данными:
- НЕ логируйте пароли и токены
- Предупреждайте пользователей о рисках
- Используйте подтверждения для критичных действий
- Сохраняйте минимум данных

## Часто задаваемые вопросы

### Можно ли запустить несколько примеров одновременно?

Нет, каждый пример управляет собственным экземпляром браузера. Запускайте примеры по очереди.

### Как остановить выполнение примера?

Нажмите `Ctrl+C` или введите команду `exit` в диалоге с агентом.

### Где хранятся логи и скриншоты?

Все логи и скриншоты сохраняются в папках `logs/` и `screenshots/` в корне проекта.

### Нужен ли интернет для работы примеров?

Да, все примеры требуют стабильного интернет-соединения для:
- Работы с API Groq
- Загрузки веб-страниц
- Взаимодействия с онлайн-сервисами
