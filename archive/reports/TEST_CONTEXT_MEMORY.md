# Тест: Проверка памяти контекста агента

## Цель теста
Проверить что агент запоминает показанные варианты и не начинает поиск заново при выборе пользователя.

## Подготовка
```bash
python main.py
```

## Тестовый сценарий №1: Заказ пиццы

### Шаг 1: Запрос блюда
```
Вы: Хочу пепперони в Красноярске
```

**Ожидаемое поведение агента:**
1. Откроет Delivery Club или Яндекс.Еду (с правильным URL!)
2. Решит капчу (если появится)
3. Введёт "пепперони" в поиск через `search_and_type`
4. Покажет список найденных вариантов

**Пример вывода:**
```
Нашёл несколько вариантов пепперони:
1. Пепперони от Красная площадь — 960₽ (600г)
2. Пицца Пепперони от Pizza epic family — 1 007₽ (650г)
3. Пепперони премиум от Симона Премиум — 1 320₽ (600г)

Какой вариант?
```

### Шаг 2: Выбор из списка
```
Вы: Пепперони премиум
```

**Ожидаемое поведение агента:**
✅ **ПРАВИЛЬНО:**
- Кликает по "Пепперони премиум" используя `click_by_text`
- НЕ открывает новый сайт
- НЕ делает новый поиск
- НЕ спрашивает город снова

❌ **НЕПРАВИЛЬНО (старое поведение):**
- Начинает navigate на другой сайт
- Делает search_and_type снова
- Спрашивает "В каком городе?"

### Шаг 3: Проверка деталей
```
Вы: Покажи состав
```

**Ожидаемое поведение:**
- Открывает карточку товара (модальное окно)
- Показывает полный состав блюда
- НЕ ищет заново

## Тестовый сценарий №2: Изменение выбора

### Продолжение от Шага 1
```
Вы: Нет, хочу первый вариант
```

**Ожидаемое поведение:**
✅ Кликает по "Пепперони от Красная площадь" (первый из показанного списка)
❌ НЕ начинает новый поиск

## Тестовый сценарий №3: Новый запрос

### Продолжение
```
Вы: Хочу бургер
```

**Ожидаемое поведение:**
✅ Понимает что это НОВЫЙ запрос (не выбор из списка)
✅ Делает новый поиск по "бургер"
✅ Показывает новый список

## Критерии успеха

### ✅ ТЕСТ ПРОЙДЕН если:
1. При выборе из списка агент **сразу кликает** по выбранному варианту
2. Агент **не спрашивает** информацию которую уже знает (город)
3. Агент **не переходит** на новый сайт при выборе из списка
4. Агент **различает** выбор из списка и новый запрос

### ❌ ТЕСТ ПРОВАЛЕН если:
1. При выборе "Пепперони премиум" агент начинает **новый поиск**
2. Агент **переспрашивает** город после того как уже искал
3. Агент **уходит** с текущей страницы при выборе из списка
4. Агент **не понимает** что пользователь выбирает из показанного списка

## Дополнительные проверки

### Проверка URL Яндекс.Еды
Если агент выбрал Яндекс.Еду, проверьте URL в логах:
- ✅ `https://eda.yandex.ru/krasnoyarsk?shippingType=delivery`
- ❌ `https://yandex.ru/eats/krasnoyarsk` (неправильный домен!)
- ❌ `https://yandex.ru/eda/krasnoyarsk` (старый URL!)

### Проверка использования search_and_type
В логах должно быть:
- ✅ `search_and_type("пепперони")`
- ❌ `type_text(selector, "пепперони")` - требует селектор

## Логирование результатов

При тестировании обращайте внимание на:
1. Какие действия выполняет агент (navigate, search_and_type, click_by_text)
2. Что агент говорит пользователю
3. Переспрашивает ли агент известную информацию
4. Правильно ли использует контекст
